探索编程的自学之旅

热爱编程，探索技术的无尽之路

语言选择

跟大多数人一样，我也是C语言入门。市面上各种各样的编程语言令人眼花缭乱。在迈入编程领域之初，我们当中很多人会纠结到底选择学习哪一门编程语言。其实对于编程语言的选择，最重要的就是参考这门语言的应用场景。

不同的编程语言有着不同的应用场景。例如，如果你对系统开发或者底层编程感兴趣，C 或 C++ 可能是更好的选择；如果你想开发网页或者移动应用程序，JavaScript 或者 Java 会更适合你；如果你对数据分析、人工智能和爬虫感兴趣，那么 Python 就是最好的选择。跟我们每个人一样，每种编程语言都有着属于它们自己的赛道，根据自己的喜好，去选择适合自己的编程语言

当然不同语言对应的各个岗位的市场需求也是不同的，有的岗位市场需求会比较大，有的就相对较小

此外，不同语言的生态系统也存在差异。比如在后端开发领域中，Java 语言拥有众多成熟的框架和库、庞大且活跃的开发者社区、丰富的学习资源，生态系统非常完善，而Go语言的生态系统相对较弱。

我个人对网站开发有浓厚的兴趣，所以在大一下学期开始了我的 Java 学习之旅，并将其作为我的主要开发语言。

同时也涉猎了多种编程语言，如 Python、C、C++和 HTML、CSS、JavaScript 等，并将它们运用于不同的项目中。

Java 技术栈 （重点）

从大一暑假开始，我就开始接触 Java 的学习了，一路磕磕绊绊，自学。从 Java 的基础语法开始，到 Java 的特性：动态代理、反射、异常处理机制、泛型、Stream 流等等，再到 JUC（Java 并发编程）、JVM（Java 虚拟机）、Java Web、MySQL 数据库。后来又接触了前沿框架，SpringBoot + SpringMVC + Mybatis SSM，又学习了 Redis 缓存数据库、Elasticsearch 分布式搜索引擎 、RabbitMQ 消息队列、Dubbo RPC 框架、SpringCloud Gateway 网关。

当然，对于基本的四大件：数据结构与算法、计算机网络、计算机操作系统、计算机组成原理，除了上课听讲之外，我会花更多的时间在网上广泛阅读优质博文和计算机相关书籍，比如说《程序是怎样运行的》、《深入浅出计算机网络》

有一个明确的目标，朝着这个目标前进，哪里不会就学什么，边学边记录，忘了就回头看，巩固复习、加深记忆，就这么简单

学习方式

学习的过程，不应该是简单而又死板的知识堆砌，走马观花，眼高手低，知其然而不知其所以然，这样只能是越学越多、越学越糊涂。我们要在学习的过程中思维发散，多反思多总结。那么有没有什么好的方法，能最高效地提升学习效率呢？做项目，开发个人项目，尝试将所学知识运用到真实的项目中去。

做项目不仅能够验收你近段时间的学习成果，更是探索你知识盲区的有效手段。

就拿我自己举例，大一学习 C/C++，做过有职工管理系统、演讲比赛系，Java 基础语法（包、类、函数）学完，我尝试写了一个零钱通系统、房屋出租系统，学完多线程我尝试写了坦克大战和黄金矿工小游戏，学完网络编程我实现了简单的餐饮管理系统，学完 JavaWeb 尝试使用 HTML + CSS 编写了静态页面，使用 Java 作交互，第一次实现了前后端交互，学完 SpringBoot + SpringMVC + Mybatis，尝试实现了一个外卖点餐平台（瑞吉外卖）。

这些知识在不断的项目开发过程中，在我的头脑中越发牢固，我开始尝试涉足那些未曾接触过的领域：我尝试开发一个交友平台，根据用户标签计算相似度，给每个用户推荐相似度匹配的用户，但不希望每次都从数据库中查找，为数据库增加压力，所以我学习了 Redis；我尝试开发一个博客平台，实现输入关键词，点击搜索后，匹配到的博文能够高亮显示，所以我学习了 Elasticseach ，实现了对海量数据的分析检索；我尝试开发一个接口开放平台，希望能集中接收客户端的请求，执行统一的安全认证、请求转发，所以我学习了Spring Cloud Gateway 网关，提高了系统安全性，便于开发维护；我希望减少重复的代码编写，减轻当前模块体量，能够像调用本地方法一样调用远程方法，所以我学习了 Dubbo RPC 框架，实现了子系统间的高性能接口调用；我希望服务端异步处理发来的请求，提升响应速度，所以我学习了 RabbitMQ 消息队列，实现了消息生产者和消费者的解耦……

在项目开发过程中，为了方便管理项目，学习了 Maven 项目管理工具。我还了解到了代码托管服务，尝试注册了 GitHub 和 Gitee 账号，学习了解了版本控制工具 Git 和简单的 git 命令，将本地代码上传至远程仓库；在项目开发完成之后，需要进行多环境配置，尝试在云服务器部署上线,所以我尝试注册了腾讯云、阿里云的账号，白嫖它们好几个月的服务器，又系统学习了 Linux 基础命令，尝试使用宝塔面板或者 Docker 容器快捷方便地部署项目，并学习使用了 Nignx ，一个高性能的、支持负载均衡和反向代理的开源 Web 服务器，了解了其相关配置。毫无疑问，在这期间，我又了解到了云服务器的购买、宝塔面板的安装配置、域名的备案流程等相关知识。

闲暇之余，我还学习了解了微信小程序的开发流程、前端前沿框架，了解学习了 React、Vue 框架的基础语法，能编写简单的页面。能够熟练使用 Ant Design Pro 开发框架快速构建企业级管理后台和前端应用，同时也了解到 Ant Design Vue、Vite 等开发框架和 Ant Design 、ElementUI、Vant 、Echarts 等优秀的组件库。

跟所有同学一样，在我的自学过程中，有过太多的第一次：第一次安装虚拟机、第一次搭建博客、第一次白嫖云服务器、第一次域名备案、第一次成功部署项目……

我学的东西很多，那么我是如何快速入门一个新知识的？我有自己的学习风格。比方说在开发一个接口开放平台时，需要用到网关实现请求转发、权限校验的功能。我首先快速地搜寻网关相关的视频资料、博文介绍，了解网关的分类、主要作用以及现有的多用来做网关的中间件有哪些。在经过多方调研之后，最终决定使用 Spring Cloud Gateway 作为网关实现。之后的工作就是结合搜罗的博文资源和阅读官方文档，学习了如何在项目中集成 SpringCloud Gateway。在使用的过程中，如果遇到了看不懂的报错，或者配置方面的问题，也可以询问当代大火的 ChatGPT 解决问题。我了解并运用了 Gateway 网关的权限校验、负载均衡、请求转发、过滤等功能，最终完成了这部分功能的开发。从基础知识的了解，到技术选型，再到编码开发，整个过程我都记录在了个人博客上，这已经成为了我最宝贵的财富。

适当参加一些考试、竞赛也是一种快速提升自己专业能力的方法。比方说我自己报名参加了今年下半年的软考中级（软件设计师），在备考期间系统地巩固学习了 23 种设计模式、软件的完整开发流程等知识。我也报名了明年的蓝桥杯赛事，也是对自己的算法学习的一种督促。

学习社区

除了前面提到过的 GitHub / Gitee 这样的开发者社区，我平时在逛的技术社区主要有：掘金、牛客、CSDN、知乎等等，这些都是我宝贵的知识源泉

打造清晰的职业路径

塑造未来，规划职业发展之路

个人学习与企业开发的接轨

前面谈到了我现阶段的专业水平能力、学习经验和学习方法，我们学习专业知识的目标，一方面是处于兴趣，另一方面无非是为了将来找好工作，进入企业，接触真实的项目开发。有志向的同学还会梦想着进入大厂深造。这就不得不谈到个人学习与企业开发的接轨了。

企业中的项目开发流程，跟个人学习阶段是完全不同的，这一点我非常清楚，这也是我们本次职业规划大赛的初衷：做好大学生就业指导工作，提高就业率。跟个人学习最大的不同的是，真实的项目开发，不仅仅要考虑到编码开发的难度，而是需要在各方面做足工作。我仅按自己的理解，浅谈一下真实的项目开发流程：

在项目开发前需要与客户充分沟通，明确项目的功能需求，制定项目计划；根据项目需求和规模，多方调研，选择合适的技术栈和开发工具；组建合适的团队，做到团队成员之间沟通明确，确保各成员各司其职、分工明确；按照项目计划编码开发，再由测试人员执行测试，确保代码质量和功能完善；准备生产环境，进行软件的部署和配置；在系统上线后，还要有专门的运维人员，负责系统监控、故障排查和性能优化，确保系统能够稳定运行。

由此可见，企业中的真实的项目开发流程是很复杂的，复杂但又不失条理，整个开发流程还是很清晰的

如何平衡技术能力和业务逻辑

到了这个阶段，我们面临的是学生到打工人的转变，是大学校园到职场生活的转变。而企业是：【】这个时候，我们就要开始考虑如何平衡技术能力和业务逻辑。企业最关心的是什么？是你能给公司带来什么，为公司创造什么价值，当然不是招你进来喝西北风的。

在工作的起步阶段，我们应该侧重于关注技术的提升，扩大技术视野和技术广度，提高自己的通用竞争力。在工作了 3年之后、5 年之后，我们积累了一定的技术经验和项目经验，这个时候更需要关注业务。我们可以在某一个细分领域下去沉淀，做业务专家。这个时候不管是做了架构师，还是做了技术管理，亦或是还奋斗在一线，最重要的是从代码中剥离出来，形成自己的业务闭环。需要把积累的业务只是知识通过互联网分享出来，多参加行业论坛和会议，并能作为嘉宾发言，多分享行业认知，积累行业人脉和资源

只有当业务知识积累到一定程度，才有可能突破职业瓶颈。过了四十岁我们写不动代码了，我们至少不用在是送外卖和摆地摊之间做选择，大不了我们可以做项目管理、做产品经理，给自己的未来多一份选择，从现在开始做起

以上只是我本人对互联网行业前景的拙见，观点可能非常不成熟，可能会闹笑话，仅代表个人观点。

做好对自己的定位

下面这张图叫邓宁-克鲁格心理效应，我们可以看到绝大多数同学处于巨婴阶段，不知道自己不知道，这是最可怕的区间，所以才会出现毕业了才意识到自己知识不够、难找工作的情况。知道自己不知道的时候，虽然信心会越来越低，但那是通往成功的必由之路。而当你知道自己懂了，你的进步将是飞快的

我今年才 20 岁，我还很年轻，我很有想法，我对未来充满想象力。而年轻就是最大的资本，时间会证明一切
