语言选择

跟大多数人一样，我也是C语言入门。市面上各种各样的编程语言令人眼花缭乱。在迈入编程领域之初，我们当中很多人会纠结到底选择学习哪一门编程语言。其实对于编程语言的选择，最重要的就是参考这门语言的应用场景。

不同的编程语言有着不同的应用场景。例如，如果你对系统开发或者底层编程感兴趣，C 或 C++ 可能是更好的选择；如果你想开发网页或者移动应用程序，JavaScript 或者 Java 会更适合你；如果你对数据分析、人工智能和爬虫感兴趣，那么 Python 就是最好的选择。跟我们每个人一样，每种编程语言都有着属于它们自己的赛道，根据自己的喜好，去选择适合自己的编程语言

当然不同语言对应的各个岗位的市场需求也是不同的，有的岗位市场需求会比较大，有的就相对较小

此外，不同语言的生态系统也存在差异。比如在后端开发领域中，Java 语言拥有众多成熟的框架和库、庞大且活跃的开发者社区、丰富的学习资源，生态系统非常完善，而Go语言的生态系统相对较弱。

我个人对网站开发有浓厚的兴趣，所以在大一下学期开始了我的 Java 学习之旅，并将其作为我的主要开发语言。

同时也涉猎了多种编程语言，如 Python、C、C++和JavaScript等，并将它们运用于不同的项目中。

个人项目

学习社区

GitHub，Gitee，非常友好的学习社区
丰富的学习社区，掘金，牛客，CSDN
BOSS直聘

在平常的编码开发中，我会将本地代码上传至 GitHub /Gitee 仓库。GitHub和Gitee都是知名的代码托管平台，它们提供了功能强大的版本控制和协作工具，开发者可以通过在这些平台上创建和托管自己的项目，展示自己的项目经验，吸引他人的关注和协作。

学会了使用版本控制工具 Git 实现代码的远程推送、拉取，了解到 一些最基础的 git 操作命令。

无论是GitHub还是Gitee，它们都成为了开发者社区和开源项目的重要组成部分。同时，这些平台也提供了丰富的社交功能，开发者可以通过关注、关注和讨论等方式，与其他开发者进行交流和互动。通过参与GitHub和Gitee社区，开发者可以提升自己的编码能力，学习和分享优秀的技术资源。

除了 GitHub / Gitee 社区，我平时在逛的技术社区主要有：掘金、牛客、CSDN、知乎等等

学习方式

对于如何快速入门一个新知识，我有自己的学习风格。比方说在开发接口开放平台时，需要用到网关实现请求转发、权限校验的功能。我首先快速地搜寻网关相关的视频资料、博文介绍，了解网关的分类、主要作用以及现有的多用来做网关的中间件有哪些。在经过多方调研之后，最终决定使用 SpringBoot Gateway 作为网关实现。之后的工作就是结合博文资源和阅读相关官方文档，学习了如何在项目中集成 SpringBoot Gateway。在使用的过程中，如果遇到了看不懂的报错，或者配置方面的问题，也可以询问当代大火的 ChatGPT 解决问题。我了解并运用了 Gateway 网关的权限校验、请求转发、过滤等功能，最终完成了这部分功能得开发。从基础知识的了解，到技术选型，再到编码开发，整个过程我都记录在了个人博客上，这已经成为了我最宝贵的财富。

适当参加一些考试、竞赛也是一种快速提升自己专业能力的方法。比方说我自己报名参加了今年下半年的软考中级（软件设计师），在备考期间系统地巩固学习了 23 种设计模式、一个软件的完整开发流程等知识。我也报名了明年的蓝桥杯赛事，是对自己的算法学习的一种督促。

个人学习与企业开发的接轨

前面谈到了我现阶段的专业水平能力、学习经验和学习方法，我们学习专业知识的目标，一方面是处于兴趣，另一方面无非是为了将来找好工作，进入企业，接触真实的项目开发。有志向的同学还会梦想着进入大厂深造。这就不得不谈到个人学习与企业开发的接轨了。

企业中的项目开发流程，跟个人学习阶段是完全不同的，这一点我非常清楚，这也是我们本次职业规划大赛的初衷：做好大学生就业指导工作，提高就业率。跟个人学习最大的不同的是，真实的项目开发，不仅仅要考虑到编码开发的难度，而是需要在各方面做足工作。我仅按自己的理解，浅谈一下真实的项目开发流程：

在项目开发前需要与客户充分沟通，明确项目的功能需求，制定项目计划；根据项目需求和规模，多方调研，选择合适的技术栈和开发工具；组建合适的团队，做到团队成员之间沟通明确，确保各成员各司其职、分工明确；按照项目计划编码开发，再由测试人员执行测试，确保代码质量和功能完善；准备生产环境，进行软件的部署和配置；在系统上线后，还要有专门的运维人员，负责系统监控、故障排查和性能优化，确保系统能够稳定运行。

由此可见，企业中的真实的项目开发流程是很复杂的，复杂但又不失条理，整个开发流程还是很清晰的
